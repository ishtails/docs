## Documentation Structure

Verity's documentation is consolidated into four primary logical chunks:

- **`introduction.mdx`**: Core value proposition and features.
- **`user-guides.mdx`**: Teacher and Learner end-to-end journeys.
- **`architecture.mdx`**: Technical stack, CRE workflows, and smart contracts.
- **`api-reference/`**: Web API endpoints and integration details.

## Local Preview

To preview your changes locally, use the Mintlify CLI:

```bash
mint dev
```

## Quality Checks

Before pushing changes, check for broken links:

```bash
mint broken-links
```

## Writing Principles

- **Concise**: One idea per sentence.
- **Visual**: Use Mermaid diagrams for flows and architecture.
- **Active Voice**: Speak directly to the user ("you").
- **Consistent**: Use **Bold** for UI elements and `Code` for file paths and commands.
