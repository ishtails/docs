## Source of truth

Primary product source files:
- `docs/verity-spec.md`
- `docs/documentation/*`
- `packages/web/src/app/demo/*`

## Updating docs

1. Update product/spec details in `docs/verity-spec.md`.
2. Sync concise docs in `docs/documentation`.
3. Mirror publish-ready content into Mintlify pages under `packages/docs/verity`.
4. Update navigation in `packages/docs/docs.json` if pages change.

## Local preview

```bash
cd packages/docs
mint dev
```

## Quality checks

```bash
mint broken-links
```

## Writing style

- Keep pages short and practical.
- Prefer diagrams for flow and architecture.
- Keep examples aligned with current demo routes.
