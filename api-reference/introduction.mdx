The Verity Web API provides endpoints for managing listings, sessions, and user profiles.

## Authentication

All API requests must include a Privy authentication token in the `Authorization` header:

```bash
Authorization: Bearer <privy_token>
```

## Base URL

```text
https://api.verity.heatiroi.xyz
```

## Core Endpoints

### Sessions
- `GET /sessions`: List all available session listings.
- `POST /sessions`: Create a new listing (Host).
- `GET /sessions/:id`: Get detailed information about a specific session.

### Meetings
- `POST /meetings`: Generate a Google Meet URL for a booked session.

### Users
- `GET /users/me`: Get current user profile and wallet info.
- `PATCH /users/me`: Update profile details.
