## Contract actions

- `requestSessionRegistration()`
- `fundSessionEscrow()`
- `requestEvaluation()`
- `claimTeacher()`
- `claimLearner()`
- `dispute()`

## API domains

- `sessions`
- `meetings`
- `goals`
- `users`
- `ai`

## Typical request sequence

1. Create session metadata and goals.
2. Register session on-chain.
3. Fund learner escrow.
4. Complete session and request evaluation.
5. Deliver AI report/oracle result.
6. Enable claims and dispute window.
