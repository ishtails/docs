## Frontend Layer
- **Stack:** React 19, TanStack Router, TanStack Query, shadcn/ui, Privy (auth/wallet), wagmi.
- **Key Routes:** `/dashboard`, `/listings`, `/listings/$id`.

## Server Layer (Hono)
- **Runtime:** Bun.
- **Database:** PostgreSQL + Drizzle ORM.
- **Responsibilities:**
  - **Meeting URL Generation**: Calls Google Calendar API to generate Meet URLs.
  - **Data Caching**: Caches on-chain data for fast dashboard loads.
  - **Auth**: Validates Privy tokens via middleware.

## Complete Session Flow

1. **Teacher** creates listing via Web UI.
2. **Server** stores metadata in DB.
3. **Learner** books session via Web UI.
4. **Server** generates Google Meet URL via Calendar API.
5. **Learner** pays USDC on-chain.
6. **Meeting** happens (Recall bot joins).
7. **CRE Workflows** take over for initiation and settlement (see [CRE & AI Workflows](/technical-architecture)).
